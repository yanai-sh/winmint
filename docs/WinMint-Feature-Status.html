Product Shape

WinMint is a Windows 11 ISO builder for clean developer workstation installs. The UI and CLI create intent, the engine services Windows media, reports explain the build, and FirstLogon finishes live-user setup after Windows Setup completes.

Core posture Opinionated, profile-driven, PowerShell-native, Windows-servicing-aware, and explicitly conservative about security, update, Store, WSL, WebView2, Defender, Firewall, WinSxS, IPv6, and other platform foundations.
Source model The user-selected ISO is the truth. There is no hidden golden image inside the repo.
Choice model No broad debloat matrix. Users choose coarse additive profile groups and a small number of real user-intent switches.
Current direction The branch is converging on complete BuildProfile.json authoring, a headless backend, auditable manifests, and resumable FirstLogon setup.

Fully Implemented

Headless CLI Backend

Done

WinMint-CLI.ps1 supports profile-backed builds, flag-built profiles, dry runs, validation, JSON output, work cleanup/listing, UUP source prep, profile generation, and elevation handoff.

Profile Contract

Done

BuildProfile.json owns source, architecture, target device, edition, identity, drivers, profile groups, desktop layers, WSL distros, editors, privacy, tweaks, and feature toggles such as features.launcher.

ISO and WIM Servicing Engine

Done

The engine extracts/mounts source media, validates architecture, selects editions, services WIM images, applies removals and registry hives, injects drivers, stages setup scripts, and emits an ISO.

Architecture Detection

Done

Architecture is inferred from ISO filename and cross-checked against WIM metadata and setup.exe. Mismatches abort before destructive work.

UUP Dump Source Prep

Done

The CLI accepts a UUP Dump conversion zip, applies one opinionated conversion policy, requires consent for network/conversion work, and then feeds the generated ISO into the normal build. If the user already has an ISO, they provide it directly.

AppX Removal Policy

Done

Minimal removes consumer, advertising, AI, communication, gaming, Microsoft-app, and common OEM trial packages by prefix. Copilot+ and Gaming groups preserve their related surfaces.

Offline Registry Tweaks

Done

Applies developer quality-of-life, Edge/start/search privacy, ContentDeliveryManager, UAC secure-desktop dimming, dark mode, file extension visibility, long paths, Sticky Keys, GameDVR, and OneDrive policy stamps without disabling core services.

Windows Setup Automation

Done

Generates and stages autounattend.xml, supports local-admin or official Microsoft-account OOBE, preserves Wi-Fi, sets edition policy, applies disk mode, and routes machine-phase setup through scripts.

Regional and DMA Handling

Done

Profiles carry target timezone, UI language, locale, keyboard, and home-location intent. -DmaInterop opts into EEA setup-region behavior and later restores the configured end-state.

Driver Strategy

Done

Supports default Windows drivers, host driver export for same-PC reinstall, and one explicit OEM driver pack for different-PC builds. Driver intent is represented in the profile.

SetupComplete Machine Phase

Done

Runs once as SYSTEM to restore Windows Update policy, create a restore point, install PowerShell/Terminal when possible, tune svchost split threshold, reduce boot timeout, preserve BitLocker protection, and stage FirstLogon.

FirstLogon Agent

Done

Runs live-user setup through idempotent modules with state tracking. It covers package managers, editors, WSL, opt-in launchers, shell layers, Windhawk, opt-in Phone Link policy, opt-in live install audit, and retry state.

Desktop Shell Layers

Done

Desktop UI layers are additive: standard Windows, Windhawk, YASB, and Komorebi can be selected independently and staged for FirstLogon installation/configuration.

Developer Setup Modules

Done

Developer group enables OpenSSH/developer-mode/remotesigned defaults. Editors and WSL distros are intentionally unselected until the user chooses them.

Launcher Choice

Done

Now opt-in through -Launcher FlowEverything, -Launcher Raycast, or features.launcher. Default is no launcher. FlowEverything configures Everything as a background index provider with tray noise minimized.

Build Reporting

Done

Produces machine-readable reports, dry-run artifacts, a setup plan, and a build manifest covering input media, selected editions, removals, features, payloads, drivers, warnings, and output paths.

In Progress

Backend Finalization and Polish

Priority 1

The top active priority is hardening the CLI/profile/engine path: public flag surface, profile contract polish, source handling, validation, release-readiness, and making sure the backend is boring and predictable before the new UI grows around it.

UUP Source Decision

In progress

Product policy: if the user already converted UUP Dump into a final ISO, they should provide that ISO directly. UUP zip input is only for cases where WinMint should prepare or validate the ISO using its own opinionated conversion policy.

Release Automation

In progress

The Cloudflare irm | iex path is configured. CI/CD should keep normal pushes and pull requests to validation only, then publish release assets only from a pushed version tag or explicit manual release run.

GPUI Product Surface

Active direction

WPF is deprecated. GPUI is the active UI direction, but should follow backend finalization rather than race ahead of it. Report and audit viewing belongs in the GPUI roadmap after profile authoring, source probing, and build supervision are stable.

Architecture/Profile Engine Migration

Ongoing

The target architecture is clear and mostly embodied in the repo. The migration remains incremental: keep the app runnable while responsibility moves toward cleaner profile, engine, reporting, FirstLogon, and GPUI boundaries.

Stubs, Labs, and Placeholders

GPUI Native App

Direction

apps/WinMint.GPUI is still isolated and early, but it is now the intended UI direction. It currently writes lab intent JSON and bridges through existing PowerShell scripts; it should mature after backend contracts are finalized.

GPUI Source Probe

Stub

Planned for the lab: path picker, file drop, async PowerShell source probe, UUP readiness warnings, and native prompts. These are roadmap items, not production behavior.

GPUI Build Console

Stub

Planned for the lab: launching CLI dry runs/builds, streaming logs, cancellation, reveal-output actions, and retry controls. Build logic must remain in PowerShell.

GPUI Manifest Viewer

Stub

Planned for the lab: load BuildManifest.json, display removed packages, warnings, staged files, source metadata, and output paths.

Terminal Wizard/TUI

Not planned

The current direction explicitly avoids a v1 terminal wizard. The CLI is a profile authoring and execution surface, not a second interactive product.

Dev Drive / Workspace Provisioning

Future

The debloat strategy mentions future developer workspace and Dev Drive handling. Today, WSL projects should live under the Linux filesystem, and Windows-native caches remain user-managed.

Container Runtime Setup

Deliberately outside

Docker/container runtimes are intentionally outside the WinMint UI. WinMint keeps WSL healthy and leaves distro-level container choices to the user.

Recurring Debloat/Maintenance

On ice

Not part of the installed system. WinMint should not leave behind scheduled tasks, services, or maintenance scripts for drift control. Once setup is done, the user owns ongoing machine management.

Personal Readout

Most solid areas

The backend contract, CLI/profile flow, DISM servicing pipeline, FirstLogon modular model, AppX policy, and validation suite are the strongest parts of the repo right now.

Most active areas

Backend finalization and polish are the number one priority. The key open decisions are public CLI surface, final UUP source behavior, release automation, and keeping GPUI aligned with the backend instead of duplicating decisions.

Things that are intentionally not bugs

No terminal wizard, no maintenance task or drift-control service, no retained maintenance script, no granular debloat matrix, no container runtime setup, no Defender/Firewall/Update/WebView2/WinSxS removal, and no automatic launcher install unless opted in.
